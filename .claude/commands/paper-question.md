description: 論文テキストに対して質問に回答する（研究のプロとして）
argument-hint: {pdf_id} {質問内容}
---

論文のプレーンテキスト（clean_text）を読み込み、研究のプロとしてユーザーの質問に分かりやすく回答します。

## 引数の確認

**受け取った引数**: `$ARGUMENTS`

引数の形式: `{pdf_id} {質問内容}`

### 引数が空の場合

以下の手順で論文一覧を取得して選択肢として提示してください:

1. **Globツール**で `clean_text/*.txt` を検索してファイル一覧を取得
2. 取得したファイルから最新10件程度のpdf_idを抽出（ファイル名から.txtを除いたもの）
3. 各pdf_idに対して **Readツール**で `memo/{pdf_id}.txt` の1行目（タイトル）を取得
4. 一覧形式でユーザーに提示

**重要**: 複雑なBashスクリプト（whileループ、$()コマンド置換など）は使用しないこと。Claude Codeのツール（Glob, Read）を直接使用すること。

その後、ユーザーに以下を質問してください：
1. どの論文について質問しますか？（PDF IDを指定）
2. どのような質問がありますか？

## 実行手順

### ステップ1: 引数の解析

`$ARGUMENTS` を解析します：
- 最初のスペースまで: PDF ID
- それ以降: 質問内容

### ステップ2: 論文テキストの読み込み

Readツールを使って論文テキストを読み込みます：

```
$PAPER_DIR/clean_text/{pdf_id}.txt
```

※ `$PAPER_DIR` は環境変数またはカレントディレクトリ

### ステップ3: タイトルの取得

メモファイルの1行目からタイトルを取得：

```
$PAPER_DIR/memo/{pdf_id}.txt
```

### ステップ4: 質問への回答

読み込んだ論文テキストに基づいて、**あなた自身（Claude）が直接回答**します。

## 回答方針

以下の方針で回答してください：

1. **正確性**: 論文に書かれていることのみに基づいて回答
2. **分かりやすさ**: 専門用語は噛み砕いて説明
3. **構造化**: 箇条書きや番号付きリストで整理
4. **引用**: 可能であれば論文内の該当箇所や概念を示す
5. **誠実さ**: 論文に書かれていない場合は「この論文では言及されていません」と明記

## 回答形式

```markdown
## 論文Q&A

**論文**: [論文タイトル]

**質問**: [ユーザーの質問]

**回答**:
[あなたの回答]
```

## 質問例

ユーザーが質問内容を指定していない場合、以下の例を提示してください：

- 「この論文の研究目的は？」
- 「使用されているデータセットは？」
- 「実験の結果を要約して」
- 「この研究の限界は何？」
- 「関連研究にはどんなものがある？」
- 「実務への応用可能性は？」
- 「著者の主張の根拠は？」
